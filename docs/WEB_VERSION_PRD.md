## 一、项目概述

### 1.1 目标
基于现有 PU2 桌面端背词工具，打造面向全网的 Web 版产品。参考 edclub.com 的视觉风格、交互体验和游戏化机制，打造一款适合小学生（7-10 岁）的趣味英语词汇学习平台。

### 1.2 核心价值主张
- **有趣**：游戏化闯关，积累徽章成就感
- **有效**：听、说、读、写、拼全链路练习
- **可传播**：社交分享、邀请好友、排行榜

### 1.3 目标用户
| 角色 | 描述 |
|------|------|
| 主要用户 | 小学生（7-10 岁），使用 Power Up 2 教材 |
| 决策者 | 家长（付费/监督学习） |
| 潜在用户 | 培训机构、学校的英语教师 |

### 1.4 需求更新（2026-04-09）
| # | 需求 | 说明 |
|---|------|------|
| 1 | 课程包架构 | 词库做成独立课程包，后续可扩展 PU1/PU3 等 |
| 2 | 纯英英释义 | 释义、例句均使用英文，不出现中文 |
| 3 | 快速出 Demo | MVP 优先跑通核心流程 |
| 4 | 英式发音 | edge-tts en-GB-SoniaNeural |
| 5 | 文件名规范 | 单词/闪卡/音频文件名统一，方便后续替换 |

### 2.1 edclub 核心特征（参考点）

### 5.3 数据模型

#### 词库结构（同桌面端）
```typescript
interface VocabUnit {
  unit: string;        // "U1"
  title: string;       // "Unit 1 - A day on the farm"
  words: VocabWord[];
}

interface VocabWord {
  word: string;       // "happy"
  definition: string; // English definition, e.g., "feeling joy and pleasure"
}
```

#### 用户进度（LocalStorage）
```typescript
interface UserProgress {
  words: Record<string, WordStatus>; // "U1:happy" -> { status: "mastered", correctCount: 3 }
  achievements: string[];            // 已解锁徽章 ID 列表
  totalScore: number;                // 总得分
  streakDays: number;                // 连续学习天数
  lastStudyDate: string;             // 上次学习日期 YYYY-MM-DD
  stats: {
    totalAnswered: number;
    correctCount: number;
    dailyHistory: DailyStat[];      // 最近 7 天
  };
}

type WordStatus = "new" | "fuzzy" | "mastered";
```

#### 文件命名规范（核心原则）
统一命名格式：`{课程编号}_{单元}_{单词}.ext`

| 资源类型 | 格式 | 示例 |
|----------|------|------|
| 词库数据 | `{course}_{unit}_{word}.json` | `pu2_u1_happy.json` |
| 闪卡图片 | `{course}_{unit}_{word}.png` | `pu2_u1_happy.png` |
| 音频文件 | `{course}_{unit}_{word}.mp3` | `pu2_u1_happy.mp3` |

**好处**：
- 定位快：知道单词就能直接找到对应文件
- 替换方便：单独替换某个闪卡/音频不影响其他文件
- 扩展简单：新课程只需按格式添加文件

## 八、盈利模式详细设计

### 8.1 免费/付费划分
```
┌─────────────────────────────────────────┐
│  Level 1 (PU1)  │ U1-U3 免费  │ U4-U9 付费 │
```

### 8.2 用户体系
- **游客**：可体验 U1-U3（限每日 N 次）
- **注册用户**：完整访问 U1-U3，解锁后访问 U4-U9
- **付费用户**：已解锁任意 Level 的全部 Unit

### 8.3 支付流程（v2）
- 微信/支付宝支付
- 支付成功后自动解锁对应 Level
- 订单记录可查
